Программа и методика испытаний (ПМИ)

1. Объект испытаний
Макет среды разработки IEC 61499 (Runtime Mock / Client Mock)
Версия: 1.0
Платформа: Windows/Linux (Python 3.10+)

2. Цель испытаний
Проверка соответствия ТЗ:
- Корректная загрузка проекта
- Обработка команд
- Устойчивость к fuzzing
- Анализ крашей

3. Требования к программе
3.1. Функциональные
- TCP deploy (deploy_client.py)
- Симуляция runtime (vulnerable_runtime.py)
- Фаззинг (fuzzer.py)
- Анализ core (analyze_core.sh)

3.2. Надёжность
Обработка ошибок без краша (кроме тестовых уязвимостей)

4. Требования к документации
README.md, tz.md, PMI.md, код с комментариями

5. Состав и порядок испытаний
Средства: Python, zzuf, gdb, pandoc
Порядок:
1. Запуск сервера и клиента
2. Загрузка тестового проекта
3. Фаззинг (100 итераций)
4. Анализ core dump
5. Генерация PDF

6. Методы испытаний
6.1. Тест 1: Запуск
Запустить vulnerable_runtime.py → "Server listening"
6.2. Тест 2: Deploy
python deploy_client.py → OK-ответы
6.3. Тест 3: Фаззинг
python fuzzer.py → краш в некоторых итерациях
6.4. Тест 4: Core анализ
./analyze_core.sh → backtrace с SIGSEGV
